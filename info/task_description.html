We are going build ideal robocity, where all buildings will be rectangular and streets are laid along south-north and east-west lines. First we need a representation for building and in this mission we will create class for this.

Because all building are rectangular and walls are parallels to streets we can define any building by coordinates of North-West corner as two arguments, size from west to east edge, size from north to south wall and height of the building. These values are described as numbers (conventional units). And we need couple operations. Father you can see class description. You should realize this.

class Building(north, west, width_WE, width_NS, height=10)

Returns a new Building instance with North-West corner at [north, west] coordinates, size is width_WE by width_NS and the height of the building is height. "height" is a positive number with default value is 10. 

>>> Building(10, 10, 1, 2, 2)
Building at [10, 10]. Size 1 by 2. Height 2.
>>> Building(0, 0, 10.5, 2.546)
Building at [0, 0]. Size 10.5 by 2.546. Height 10.

corners()

Returns a dictionary with coordinates of the building corners. The dictionary has next keys: "north-west", "north-east", "south-west", "south-east". The values are tuples with two numbers. 

>>> Building(1, 2, 2, 2).corners()
{"north-west": (1, 2), "north-east": (1, 4), "south-west": (3, 2), "south-east": (3, 4)}

area()

Returns the area of the building. 

>>> Building(1, 2.5, 4.2, 1.25).area()
5.25

volume()

Returns the volume of the building. 

>>> Building(1, 2.5, 4.2, 1.25, 101).volume()
530.25

__str__()
__repr__()

String representation of the Building. This method is using for "print" or "str" conversation. Returns the string in the next view:
"Building at [{north}, {west}]. Size {width_we}, {width_ns}]. Height {height}."

>>> str(Building(0, 0, 10.5, 2.546))
"Building at [0, 0]. Size 10.5 by 2.546. Height 10."
